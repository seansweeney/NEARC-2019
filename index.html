Migrating AGOL User Accounts Using the ArcGIS API for Python

Sean Sweeney, GIS Senior Programmer Analyst

City of Cambridge, MA

NEARC - Newry, ME

October 21, 2019

Cambridge is moving the majority of our ArcGIS Online users from AGOL logins to enterprise logins. Our goal is to move users in a controlled manner - not only their content, but their profile and other aspects of their AGOL identity - from their old AGOL login to their new enterprise account. I'll demonstrate some of the components of the ArcGIS API for Python we're using to accomplish this, point out a few curveballs we've run into along the way, and also show final Python script that (mostly) automates the process.

Role

if (agol_user.role == agol_user.roleId):
		# Built-in user role - assign directly
		sso_user.update_role(role=agol_user.role)
	else:
		# Custom role - first get role object from RoleManager
		role = gis.users.roles.get_role(agol_user.roleId)
		sso_user.update_role(role=role)
For built-in user roles (org_user, org_publisher, org_admin, viewer, view_only, viewplusedit) you can just pass the role property to update_role. Users that don't have a custom role will have User.role == User.RoleId.

For custom roles you have to get the role object from the RoleManager static class (via gis.users.roles).

You can't just use the second method for all of them because the built-in roles don't exist in the RoleManager. 
if (agol_user.role == agol_user.roleId):
		# Built-in user role - assign directly
		sso_user.update_role(role=agol_user.role)
	else:
		# Custom role - first get role object from RoleManager
		role = gis.users.roles.get_role(agol_user.roleId)
		sso_user.update_role(role=role)

print('Updating groups'.center(40,'-'))
for group in agol_user.groups:
	if (group.owner == agol_username):
		print('Changing ownership: ' + group.title)
		group.reassign_to(sso_username)
	else:
		print('Joining: ' + group.title)
		group.add_users([sso_username])
